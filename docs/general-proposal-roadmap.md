# See-Sweet General Proposal Roadmap

See-Sweet should be positioned as a private executive secretariat for individuals who need leverage, clarity, and continuity. The product ambition is not to offer another AI chat interface, but to give each user the practical support structure that high-performing CEOs usually build around themselves through chiefs of staff, executive assistants, and operating cadences.

## Plan

### Product Direction

The core proposal is that every user should be able to operate with the leverage of a disciplined executive office. See-Sweet should function as a private layer for preparation, prioritization, memory, coordination, and follow-through. Its value is not novelty; its value is dependable operational support that reduces cognitive load and raises execution quality across the user's professional life.

### Use Cases

1. A founder uses See-Sweet to prepare for board and investor conversations, retain decision context, and keep follow-through visible across strategy, hiring, and fundraising.
2. A senior executive uses See-Sweet as a private operating desk for daily briefs, meeting preparation, stakeholder context, and structured decision support.
3. A consultant, advisor, or solo operator uses See-Sweet to run multiple client and research streams without losing continuity, commitments, or context between sessions.

### Core Features

1. Persistent assistant memory scoped to the authenticated user and their working history.
2. A daily executive briefing with agenda signals, deadlines, blocked items, and unresolved decisions.
3. Meeting prep packs with objectives, participant context, historical notes, and suggested talking points.
4. Post-meeting capture that turns conversations into summaries, actions, owners, and deadlines.
5. Task and priority orchestration linked to notes, meetings, and assistant conversations.
6. Contact and relationship intelligence with interaction history and relevant context.
7. Decision logging with rationale, timing, current status, and traceable change history.
8. Retrieval-backed knowledge access across documents, notes, and stored context.
9. Configurable assistant behavior so each user can shape tone, workflow style, and proactive guidance.
10. Multilingual support for international and cross-border executive workflows.

## Done

This proposal establishes the first positioning baseline for discussion:

- See-Sweet is framed as a personal executive secretariat, not a general-purpose chat tool.
- The value story is operational leverage: preparation, memory, coordination, and disciplined follow-through.
- The initial roadmap favors a strong assistant core before expanding into more autonomous workflow execution.

## Later

The roadmap can be discussed in four stages:

1. Foundation: reliable chat, user memory, document retrieval, assistant configuration, and multilingual support. ✅ Done
2. Notification system: real-time LLM-generated notifications, persistent inbox, SSE delivery, scalable category model. ✅ Done
3. Knowledge base: document upload + RAG retrieval, personal info record, context slot registry. ✅ Done
4. Executive workflow layer: contacts, meetings, tasks, assistant-driven creation, reports, and home briefing. ✅ Done (feat/core-4)
5. Chief-of-staff layer: decision log, LLM briefing, meeting prep packs, post-meeting capture, contact interaction history, and proactive nudges. ✅ Done (feat/core-5)
6. SaaS launch layer: billing (Stripe), email (Resend), admin panel, rate limiting, and onboarding flow. 🔲 Planned (phase-6-saas-launch.md)
7. UI/UX & polish pass — cross-cutting quality improvements and 24-hour Contact Card share link. ✅ Done (feat/refactor-2 — see feat-refactor-2.md)
